  B. ON-PREM SHAREPOINT SITE → SPO SITE

WHAT THE SCRIPTS DO:
  1. Bootstrap: Ensure required modules via the script's Ensure-Module helper. EXACT module list (don't invent): Microsoft.Online.SharePoint.PowerShell (SPO admin), Microsoft.SharePoint.MigrationTool.PowerShell (SPMT), PnP.PowerShell (Connect-PnPOnline -UseWebLogin), ActiveDirectory (RSAT-AD; for OD2OD post-migration AD block). Pre-launch SPMT executable to populate DLLs; load SP CSOM DLLs via Add-Type. OD2OD DLL path: F:\Tools\ ; SP2SPO DLL path: F:\IAU_Scripts\OneDrive_Migration_Scripts\
  2. Get-Credential prompts for SPO admin AND on-prem SP admin. Scripts MUST run interactively.
  3. Connect-PnPOnline to the driver list site.
  4. Pull list items where Migrate = "Migrate" (literal — NOT "Ready"; operator sets the column to "Migrate" to trigger).
  5. For each item: (a) set lock-flag to "Processing" so other runners skip — column differs by flow: OD2OD = "Inprocess", SP2SPO = "Processing"; (b) Register-SPMTMigration with SPO + on-prem credentials; (c) Add-SPMTTask (OD2OD source/target or SP2SPO source/target -MigrateAll); (d) Start-SPMTMigration; watch status.
  6. Status written to list "Migrate" column: Success → Migrated, Log = <path>. Per-file errors → ErrorLog, FailureSummaryReport.csv attached (if file exists). NO email is sent by the script — a Power Automate flow on the list emails the user. Hard failure → Failed.
  7. Post-migration (OD2OD only, ON SUCCESS): SCA swap on the user's SOURCE on-prem MySite ($sourceUrl, NOT the new SPO OneDrive) — ensure svc-migration, make it Site Collection Admin on the source, demote the migrated user; this locks down the source. AD updates RUN by default: AD wwwHomePage → new SPO OneDrive URL, remove user from all *REDIRECTION* groups and SecFltr-USR-OneDrive, add to SecFltr-USR-Office365. NOTE: the `#<#` / `#>` markers around the AD block LOOK like a block comment but PowerShell treats `#<#` as a single-line comment (leading `#` comments the whole line, so `<#` is just text) — the AD code actually executes. If asked "does it update AD?": "Yes, by default. To truly disable, change `#<#` to `<#` (remove the leading `#`)."
  8. Sleep between items: OD2OD 30s; SP2SPO 120s.
  9. Loop until the list is exhausted; Stop-Transcript.

DRIVER LIST COLUMNS (key): SourceUrl, TargetUrl, Migrate (Ready/Migrating/Migrated/Failed/ErrorLog), Log, StartDate, CompletedDate. Lock flag: Inprocess (OD2OD) OR Processing (SP2SPO). NO TimeZone column here (that's a Common Drive feature).
