TWO TARGET FLOWS — always confirm which first (Global.CDFlow):
FLOW A — UNC → TEAMS CHANNEL: target is a Teams channel (a SPO site under the hood). Update-MigrationTargets.v2.ps1 (Graph) resolves TeamName → SiteUrl, TeamChannel0 → channel folder; auto-provisions the folder if uninitialized; grants svc perms; then SPMT-Worker.v2.ps1.
FLOW B — UNC → STRAIGHT SPO SITE (no Team): TeamName blank, direct SiteUrl. Skips Graph/Team resolution and channel auto-provisioning; grants perms on the site; then SPMT-Worker.v2.ps1.

UPDATE-MIGRATIONTARGETS.v2.ps1 — TWO PHASES:
Phase 1 (INTERACTIVE, DELEGATED Graph, human admin; Flow A only): resolve TeamName → SiteUrl, TeamChannel0 → channel folder; add svc account as M365 Group Owner; auto-provision channel folder; write SiteUrl back; set TeamChannelError on failure.
Phase 2 (AUTOMATED, app-only SPO Admin): storage capacity check + auto-downgrade; grant svc-migration as Site Collection Admin; idempotent. Run Phase 1 first, then Phase 2 (scales unattended).

DRIVER LIST: CommonDriveMigration on https://contoso.spo.microsoft.scloud/sites/000001
User columns: DIV, Title, ITDistro, SourcePath, TeamName, TeamChannel0, TimeZone (EST/CST/MST/PST/AKST/HST/ANYTIME), Priority (1=highest), ExtendedHours (Yes/No).
Auto-set: TeamChannelError, resolved SiteUrl, Migrate, Stage, ClaimedBy, ClaimedAt, QueuedAt, Size3YrMB/5YrMB/7YrMB, YearUsed, timestamps/log paths.

MIGRATION STATES (Migrate column): Blank (untouched), Ready (operator go), Stage (staging copy), Staged (ready for cutover), StagedWithErrors, MigrateOnly (skip staging), Migrating, Migrated, ErrorLog (errors+report), Failed.

SCHEDULING (-UseScheduling): only picks rows whose window is open. Windows: weekdays 5 PM → 6 AM (local to TimeZone); weekends + US holidays 24h. ANYTIME or ExtendedHours=Yes bypasses the window. Large Migration Threshold: rows ≥10 GB restricted to weekends/holidays (unless ExtendedHours). QueuedAt = FIFO within a priority band.

STORAGE AUTO-DOWNGRADE (Phase 2): Size3YrMB/5YrMB/7YrMB = files modified in last 3/5/7 yrs. Tries to fit destination quota 7yr → 5yr → 3yr; horizon stamped in YearUsed; SPMT filters source by last-modified date. If even 3yr won't fit, flag for manual review.

MULTI-SERVER CLAIM LOCKING: runs on multiple servers. ClaimedBy = $Env:COMPUTERNAME; ClaimedAt = claim time; ClaimStaleHours (default 2h) before a claim is abandoned and auto-released.

AUTH: certificate-based app-only ($AppClientId/$AppTenantId/$CertThumbprint per script; overridable via -AppClientIdParam / -AppCertThumbprintParam). Phase 1 uses DELEGATED Graph (interactive admin) to add the svc account as Team owner.

APP REGISTRATIONS (36 total, NOT "36 SPO apps"): 1 Graph + 6 SPO Admin (1/server) + 18 SPMT workers (3/server × 6) + 11 helper/reporting.
