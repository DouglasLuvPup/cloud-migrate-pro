On-Prem → SharePoint Online

on-prem SharePoint Server & MySites → SPO + OneDrive
  • Two flows in one playbook — MySite → OneDrive (with full AD & SCA cutover) and on-prem site → SPO site (content-only). Version history is preserved end-to-end and a configurable extension blocklist screens out risky file types.
  • List-controlled, multi-runner safe — driven by a customer-owned SPO list; only rows the operator marks Migrate are picked up, and a per-row lock plus server-name stamp lets several servers run side-by-side without ever double-processing an item.
  • SCA swap + AD cutover — promotes the migration service account to Site Collection Admin on the source MySite and demotes the user, redirects wwwHomePage to the new OneDrive URL, removes legacy redirection / OneDrive security groups, and adds the target Office 365 group.
  • Audit trail + user notifications — every row gets DateStarted, DateMigrated, log path, and a final status (Migrated / ErrorLog / Failed); the SPMT failure summary is attached to the row, and a Power Automate flow emails the affected user automatically when errors are logged.

H: Drive → OneDrive

UNC \\server\users\<sam> → OneDrive /Documents/HDrive
  • OneDrive auto-provisioning — resolves SAM → UPN → OneDrive URL and provisions the personal site with retry & backoff, then grants the dedicated migration service account site-collection admin so SPMT can write.
  • AD-aware cutover — stamps wwwHomePage to the new OneDrive URL and removes the user from every group listed on the row's redirection field (multi-group, comma- or newline-separated); license-group membership is validated up front so unlicensed users are skipped automatically.
  • Postpone-by-date & throttle-aware — operators can defer any user by date (multiple field-name spellings accepted), credentials are DPAPI-cached, and SPO 429 / 503 throttling triggers exponential backoff at every layer.
  • Source lockdown, reorg, and full audit — a stage pre-pass copies content with no AD changes, the H: source is flipped read-only in a background job, /My Documents is flattened into /Documents, .pst is blocked, and per-item logs plus failure / fatal-error reports are attached to each SPO row for sign-off.

Common Drive → Teams / SharePoint

UNC shared drives → Teams channel folder or SPO site
  • Two target flows — Flow A resolves a Team + Channel to its SPO URL via Graph (app-only), pre-grants Group Owner + Site Collection Admin, and stamps storage quota / used / available so the engine knows the row will fit before it runs. Flow B targets a SharePoint site directly.
  • Storage auto-downgrade — a parallel single-pass UNC scan sizes 3 / 5 / 7-year footprints, then auto-trims the migration horizon (7 → 5 → 3 years) to fit the destination quota with a safety buffer; the chosen horizon is stamped on the row.
  • Timezone-aware scheduling — per-row windows block daytime business hours and allow nights, weekends, and US federal holidays in the row's local time; large rows (≥ 10 GB) are held to weekends/holidays unless explicitly opted into extended hours.
  • 6+ server scale, post-migration cleanup, and reporting — atomic multi-server claim locking with stale-release across scan, target, and migrate; ACLs are flipped to read-only and the source is deleted with a per-row deletion report; an HTML pipeline dashboard summarises progress by division for leadership.

Built In, Across Every Playbook

Sovereign by design, portable by default

These are the foundations every playbook shares — the reasons they're trusted in the most regulated environments in the world.

🌐

Sovereign-cloud native — commercial-ready

Engineered for IL5 (GCC-High / DoD) and IL6 (microsoft.scloud) with the same codebase — and just as portable to commercial M365 tenants with a simple host-suffix swap. Build once, run anywhere.

🔐

Audit logging in every run

Each script writes granular per-item status to a customer-owned SharePoint driver list — Migrate / Processing / Log / StartDate / CompletedDate plus categorized failure codes (LICENSE, UPN, PROVISIONING, ACCESS, THROTTLE). Per-file CSV reports (ItemReport_R1, FailureSummaryReport, FatalError_*) attach to each row so every file is accounted for, and flat-file logs forward cleanly to Splunk / Sentinel / any SIEM. Every line is PowerShell you can diff in git — no opaque binaries, no vendor cloud in the path.

💸

Zero license cost

The engine is Microsoft's SPMT (shipped free). The orchestration, scheduling, dashboards, and AD integration are owned IP bundled with the playbooks — no per-seat or per-GB vendor fees.
