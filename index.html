2024 Playbook

On-Prem → SharePoint Online

on-prem SharePoint Server & MySites → SPO + OneDrive
  • Two flows, one playbook — Migration-OD2OD-SPO.ps1 (MySites → SPO OneDrive) and Migration-SP2SPO.ps1 (on-prem sites → SPO sites); covers SharePoint 2013 → Subscription Edition.
  • SPMT-driven, list-controlled — pulls rows marked Ready from a customer-owned SPO driver list, then Register / Add / Start-SPMTMigration with both SPO and on-prem admin credentials.
  • Multi-runner safe — a per-row lock flag (Inprocess / Processing) lets several servers run at once without ever double-processing an item.
  • SCA swap at cutover — promotes a dedicated migration service account to Site Collection Admin and demotes the migrated user; optional AD wwwHomePage redirect to the new OneDrive.

2025 Playbook

H: Drive → OneDrive

UNC \\server\users\<sam> → OneDrive /Documents/HDrive
  • OneDrive auto-provisioning — resolves SAM → UPN → OneDrive URL and provisions via Request-SPOPersonalSite (3× backoff), then grants a dedicated migration service account.
  • AD group flips on success — removes SecFltr-USR-OneDrive, adds SecFltr-USR-Office365 + the license group; multi-group RedirectGP supported.
  • Postpone-by-date & throttle-aware — defer users by date (6 accepted spellings), DPAPI-cached creds, and SPO throttling auto-retry with backoff.
  • Source lockdown + reorg — sets the H: source read-only in a background job and flattens /My Documents → /Documents after a verified move. Blocks .pst.

2026 Playbook

Common Drive → Teams / SharePoint

UNC shared drives → Teams channel or SPO site
  • Two target flows — Flow A auto-provisions a Teams channel folder via Graph (two-phase Update-MigrationTargets.v2.ps1); Flow B goes straight into a SharePoint site.
  • Storage auto-downgrade — Invoke-UNCStorageScan sizes 3/5/7-year footprints, then fits the destination quota by trimming 7 → 5 → 3 years (horizon stamped in YearUsed).
  • Timezone-aware scheduling — per-row windows: weekday nights, weekends & US federal holidays; rows ≥ 10 GB held to weekends unless ExtendedHours.
  • 6+ server scale — multi-server claim locking (ClaimedBy / ClaimedAt / stale-release); Retry-FailedMigration reprocesses failed items and 0-byte uploads.

Built In, Across Every Playbook

Sovereign by design, portable by default

These are the foundations every playbook shares — the reasons they're trusted in the most regulated environments in the world.

🌐

Sovereign-cloud native — commercial-ready

Engineered for IL5 (GCC-High / DoD) and IL6 (microsoft.scloud) with the same codebase — and just as portable to commercial M365 tenants with a simple host-suffix swap. Build once, run anywhere.

🔐

Audit logging in every run

Each script writes granular per-item status to a customer-owned SharePoint driver list — Migrate / Processing / Log / StartDate / CompletedDate plus categorized failure codes (LICENSE, UPN, PROVISIONING, ACCESS, THROTTLE). Per-file CSV reports (ItemReport_R1, FailureSummaryReport, FatalError_*) attach to each row so every file is accounted for, and flat-file logs forward cleanly to Splunk / Sentinel / any SIEM. Every line is PowerShell you can diff in git — no opaque binaries, no vendor cloud in the path.

💸

Zero license cost

The engine is Microsoft's SPMT (shipped free). The orchestration, scheduling, dashboards, and AD integration are owned IP bundled with the playbooks — no per-seat or per-GB vendor fees.
